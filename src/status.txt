CONFIRMED FIXES (this session)
==============================

1. Bank 4 -> Bank 5 overflow (decode.asm)
   CS:IP crossing $10000 from bank 4 into bank 5 (BIOS ROM).
   Fixed: detect overflow, switch to attic code cache.

2. INT 10h AH=00 BDA write bug (io.asm)
   Was writing $49 to BDA instead of video mode value.

3. BDA repair after boot (decode.asm)
   IBMBIO.COM REP MOVSW overwrites BDA. One-time repair fires at
   tick_counter+1 >= $10. Must clear $8FEF at startup.
   IMPORTANT: tick_counter+1 must be incremented or repair never fires.

4. INT 21h AH=09/02/06 intercept (opcodes.asm)
   DOS output path doesn't reach screen via IVT. We intercept:
   - AH=09: Print $ terminated string from DS:DX
   - AH=02: Write character in DL
   - AH=06: Direct console I/O (output when DL != FF)
   Uses $8FD0/$8FD1 as offset counter for AH=09.

5. POP r/m16 opcode $8F (opcodes.asm, tables.asm, decode.asm)
   Standard 8086 opcode was missing. FreeDOS kernel uses it.

6. ROM write protection (mem.asm, modrm.asm)
   Guest code was overwriting BIOS ROM (bank 5). Writes to $F0000+
   are now silently discarded in mem_write8, mem_write16, write_rm8,
   write_rm16.

7. MDA support (mem.asm, io.asm)
   - $B0000-$B7FFF now maps to same bank 2 buffer as CGA ($2A000)
   - Port $3BA (MDA status) returns toggling retrace bits like $3DA

8. INT 13h AH=03 write sectors (disk.asm)
   Writes guest memory to floppy image in attic via DMA.

9. INT 13h AH=04/15 and hard disk stub (disk.asm)
   - AH=04: Verify sectors (always succeeds)
   - AH=15: Get disk type (consistent with AH=08 for HD)
   - Hard disk (DL >= $80): AH=08 returns 1 drive, AH=02 returns
     blank sector so FreeDOS gets "illegal partition table" path

10. Code cache bank 4 vs attic source (decode.asm)
    When code cache loads a page, detect whether linear address is
    in bank 4 (< $10000, MB $00) or attic (>= $10000, MB $80).
    Fixes segments straddling the $10000 boundary.

11. Code cache invalidation on attic DMA (disk.asm)
    When INT 13h DMAs directly to attic, invalidate code cache
    to prevent stale code execution.

12. Variable collision fixes (zeropage.asm, opcodes.asm, io.asm, int.asm)
    - pusha_saved_sp: $8F50 -> $8F70
    - IDIV sign flags: $8F52/$8F53 -> $8F72/$8F73
    - Teletype debug ring: $8F50-$8F57 -> $8F74-$8F7B (idx $8F7C)
    - Removed unused STACK_TEMP ($8FE8)
    The IDIV/debug ring collision was a REAL BUG that could corrupt
    division results when printing happened during IDIV.

13. INT 8 timer delivery (decode.asm)
    - int8_asap flag set every ~4096 instructions after BDA repair
    - Delivery in opcode_done (after instruction completes, correct
      8086 interrupt timing)
    - check_hw_interrupts checks flag_if before firing
    NOTE: Higher INT 8 frequency (~256 instructions) causes garbled
    text output. Root cause unknown. Kept at ~4096 for stability.

14. Segment wrapping (mem.asm, int.asm, modrm.asm)
    Offset $FFFF->$0000 wrap detection. Unproven but correct per spec.

15. INT 13h direct DMA to attic (disk.asm)
    For destinations >= $10000, DMA directly from SECTOR_BUF to attic
    instead of byte-by-byte cache writes. Flushes and invalidates
    data and code caches before DMA.

KNOWN BUGS
==========

1. FreeDOS hangs after "sector" during InitDisk
   - Kernel needs INT 8 timer delivery during mid-boot
   - INT 8 is gated by BDA repair flag ($8FEF), which fires at
     tick_counter+1 >= $10 (~1M instructions)
   - The "sector" hang happens BEFORE the BDA repair fires
   - Enabling INT 8 earlier causes boot hang (freezes at "FreeDOS")
   - Root cause of early INT 8 hang is unknown
   - Possible fix: find a better gate than BDA repair flag

2. INT 8 at high frequency garbles text
   - At ~256 instruction interval, output text gets corrupted
   - At ~4096 interval, text is clean
   - Root cause unknown — possibly do_sw_interrupt clobbers state
     used by chrout_safe or the INT 29h handler
   - Needs investigation: which ZP variables are shared

3. BASICA DS=$0018 corruption (VIDEO_MODE=3)
   - See previous session notes
   - reg_ds is at ZP $18 — something may write to that address

4. Attic cache data loss (general)
   - Cache flush/verify passes isolated test
   - During emulation, data written through cache sometimes lost
   - Partially mitigated by direct DMA in INT 13h
   - May be related to bug #2 (state clobbering)

5. EDLIN file write bug
   - Creates file in directory but writes empty content
   - INT 13h AH=03 is implemented but may have issues
   - Needs investigation

IMPLEMENTATION STATUS
=====================
All standard 8086 opcodes implemented except:
  $9B (WAIT) — NOP, harmless
  $D8-$DF (FPU ESC) — NOP with ModR/M consumed, correct

80186 opcodes:
  $60/$61 (PUSHA/POPA) — implemented
  $68/$6A (PUSH imm16/imm8) — implemented
  $C0/$C1 (shift imm8) — implemented
  $69/$6B (3-operand IMUL) — NOT implemented, would desync IP
  $62 (BOUND) — NOT implemented
  $C8/$C9 (ENTER/LEAVE) — ENTER skips 3 bytes, LEAVE is NOP
